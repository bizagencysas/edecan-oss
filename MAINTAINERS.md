Repository permissions are the operational source of truth. This file is the public governance record and should be updated in the same change whenever maintainer responsibilities change.

## Expectations

Maintainers are expected to:

- Protect users, contributors, credentials, release infrastructure, and private data
- Review changes in proportion to risk and preserve backward compatibility
- Keep decisions transparent when security and privacy permit
- Disclose conflicts of interest and recuse themselves when appropriate
- Apply the Code of Conduct consistently
- Keep their GitHub account protected with strong authentication

## Changes to the team

The nomination, scope, inactivity, and removal processes are defined in [GOVERNANCE.md](GOVERNANCE.md). Changes to this file should be reviewed like other governance changes.

For vulnerabilities, follow [SECURITY.md](SECURITY.md). Do not disclose security reports in public issues or pull requests.
