# Support

Edecan is an open-source project maintained on a best-effort basis. This policy explains where to ask for help and what information makes a request actionable.

## Before requesting help

1. Read the README and the documentation relevant to your installation or platform.
2. Search existing issues for the exact error and the operation that triggered it.
3. Reproduce the behavior on a supported, unmodified checkout when practical.
4. Collect the Edecan version or commit, operating system, architecture, runtime versions, and exact sanitized command output.

## Where to ask

- **Reproducible defect:** Use the bug report form in the GitHub issue chooser.
- **Focused setup or usage question:** Use the support question form.
- **Product or architecture proposal:** Use the feature request form.
- **Documentation problem:** Open a bug report and select `Documentation` as the affected area.
- **Security vulnerability:** Follow [SECURITY.md](SECURITY.md). Never open a public issue.
- **Conduct concern:** Follow the private reporting guidance in [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md).

Please keep one problem per issue. Maintainers may redirect, close, or convert requests that belong in a different channel.

## Protect sensitive data

Never post credentials, access tokens, private keys, personal data, document contents, production database records, or unredacted logs. Replace sensitive values with descriptive placeholders and verify screenshots and attachments before uploading them.

If sensitive information is posted, revoke or rotate the affected credential immediately. Editing the public post is not sufficient because copies may persist.

## Scope and response expectations

Community support does not include guaranteed response times, private consulting, custom deployments, data recovery, or operational access to a user's systems. Maintainers prioritize security, regressions, reproducible defects, and work that benefits the broader project.

A complete report can still require follow-up or remain unresolved when maintainers cannot reproduce it. Requests may be closed when required information is not provided after follow-up or when the requested behavior is outside project scope.

Commercial support is not currently advertised by this repository.
